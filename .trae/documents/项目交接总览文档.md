## 一、项目概述

### 1.1 项目名称
**AI Tool** — 企业级 AI 图像/视频生成平台 + 产品知识库管理系统

### 1.2 项目定位
为公司内部搭建的综合性业务工具平台，包含两大核心模块：

| 模块 | 说明 |
|------|------|
| **AI 图像/视频生成** | 调用第三方 AI 模型聚合平台（[dmxapi.cn](https://www.dmxapi.cn)）的 API，提供文生图、图生图、文生视频能力 |
| **产品知识库管理系统** | 六层（L1-L6）产品元数据管理，支持 Excel 批量导入、草稿审批发布流程、团队协作权限 |

### 1.3 当前状态
- ✅ 后端核心 API 全部实现
- ✅ 前端页面基本完成
- ✅ AI 生成（文生图/图生图）可用
- ✅ 产品 CRUD + Excel 导入可用
- ✅ 用户/团队权限体系可用
- ⚠️ 视频生成接口预留，未实现实际调用
- ⚠️ 尚未配置 CI/CD、自动化测试、Docker 部署
- ⚠️ 使用 SQLite 开发，上线需切换 PostgreSQL

## 四、数据库设计

### 4.1 数据模型总览（12 张表）

```
┌──────────────┐     ┌──────────────┐     ┌──────────────────┐

       ├──────────────────────────────────────────────┐

       ▼                                              ▼
┌──────────────┐                            ┌──────────────────┐
│ generations  │                            │  product_drafts  │

### 4.2 L1-L6 产品数据层级说明

| 层级 | 表名 | 内容 | 示例字段 |
|------|------|------|----------|
| **L1** | `products` | 产品基本信息 | SKU、条码、品牌、系列、名称（中/英/日）、品类、等级、生命周期、上市日期、渠道、销售区域、负责人 |
| **L2** | `product_specs` | 规格参数 | 尺寸线、毛重、材质、表面处理、主色、热源、功率、容量、认证、技术优势、使用说明 |
| **L3** | `product_business` | 商务/市场信息 | 核心卖点、目标人群、差异化、价格定位、技术优势、情感价值、使用场景、竞品 |
| **L4** | `product_content` | 内容/文案/SEO | Amazon标题、网站标题、五点描述、各语言Listing、A+内容、搜索关键词、QA问答、评论标签、差评应对、共享内容SKU |
| **L5** | `product_media` | 视觉素材 | 30+ 素材字段：白底图、场景图、详情图、尺寸图、渠道素材、社交媒体素材、参考素材、禁用品类 |
| **L6** | `product_prompts` | AI 提示词模板 | 模板名称、提示词文本、参数配置、版本号、使用次数 |

### 4.3 关键表结构

**users 表：**
- `id` (UUID, PK)
- `username`, `email`, `hashed_password`
- `role`：`super_admin` / `user`
- `is_active`: boolean

**groups 表：**
- `id` (UUID, PK)
- `name`, `description`
- `is_preset`：预设"产品团队"和"设计团队"

**user_groups 表（多对多关联）：**
- `user_id` (FK → users)
- `group_id` (FK → groups)
- `group_role`：`admin` / `member`

**system_config 表（K-V 存储）：**
- `key` (PK, 字符串)
- `value` (TEXT)
- 模型配置存储格式：`key = "model_{model_id}"`, `value = "模型名称|api_format|api_key_path|description"`（管道分隔）

## 八、AI 生成工作流详解

### 8.1 支持的模型

模型通过 [dmxapi.cn](https://www.dmxapi.cn) 聚合平台调用，支持两种 API 格式：

| 模型 ID | 显示名称 | 类型 | API 格式 |
|---------|----------|------|----------|
| `gpt-image-2-ssvip` | GPT Image 2 SSVIP | 图片生成 | OpenAI 兼容 |
| `gpt-image-2` | GPT Image 2 | 图片生成 | OpenAI 兼容 |
| `gemini-3.1-flash-image-preview` | Nano Banana 2 | 图片生成 | Gemini 原生 |

### 8.2 文生图流程

```
用户输入 prompt + 参数 → 前端 POST /api/generation/txt2img
    → 后端创建 Generation 记录 (status=processing)
    → 根据模型 api_format 选择调用方式：
        ├── openai 格式：POST dmxapi/v1/images/generations
        │   └── 多张图时并行请求
        └── gemini 格式：POST dmxapi/v1beta/models/{model}:generateContent
    → 解析响应中的 Base64 图片数据
    → 保存到 /uploads/generated/ 目录
    → 更新 Generation 状态为 completed/failed
```

### 8.3 图生图流程

```
用户上传参考图(1-4张) + 可选 prompt → 前端 FormData POST /api/generation/img2img
    → 后端接收图片，自动压缩至 768px + JPEG 60% 质量
    → 调用 dmxapi/v1/images/edits (OpenAI) 或 Gemini 端点
    → 内置重试机制：408/429/5xx 自动重试 2 次
    → 保存结果图片，返回 URL
```

### 8.4 视频生成

当前接口已预留（`POST /api/generation/txt2vid`），但 `generation_service.py` 中实际调用逻辑未实现，直接返回 `status=failed`。

### 8.5 重试与超时配置

在 `dmxapi_service.py` 中实现：
- **重试**：遇到 408/429/5xx 状态码自动重试，最多 2 次
- **超时**：通过环境变量配置
  - `DMXAPI_TXT2IMG_TIMEOUT=300`（文生图 5 分钟）
  - `DMXAPI_IMG2IMG_READ_TIMEOUT=1200`（图生图读取 20 分钟）
  - `DMXAPI_IMG2IMG_CONNECT_TIMEOUT=60`（图生图连接 1 分钟）

```
